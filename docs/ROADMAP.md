# Roadmap

Agreed development roadmap, driven by the two design issues:

- [#6](https://github.com/pdlourenco/adigator-embedded/issues/6) — variable
  (runtime-free) loop bounds: Tier 0 (vectorized mode, exists), Tier 1
  (Nmax generation + runtime trip count), Tier 2 (symbolic N).
- [#11](https://github.com/pdlourenco/adigator-embedded/issues/11) — N-D
  array support: Level 1 (folded-2D/cell patterns, docs), Level 2 (N-D
  parameter veneer with affine folded windows), Level 3 (full N-D `cada`).

The combined headline use case: **optimal allocation over time** — `N`
actuators × `K` time steps, both free at runtime from one generated
derivative file, with failure/reconfiguration masks.

Foundations already in place (see `docs/ANALYSIS.md` §1.5 for the full
disposition log): all analysis findings fixed or documented-benign with
pinning tests; CI through Phase 4 (PR gate + extended suites + ratchets);
embed modes verified equivalent and codegen-compatible; literal scatter
indices, data dedup/range compression in the inline emitter.

**Active frontier (R10–R27).** R1–R9 are landed bar the R6 decision gate.
**R13 (documentation) is done:** README (#58) and user guide (#59) merged;
the two policy calls are decided — **(3) fork-versioning** and **(4) defer the
paper** (issue #18). **R10 is done** — both (a) the rolled-`for…end`-in-a-
multi-subfunction-file slice and (b) the `collapsed>0` driver assertion.
**R11 is done** (ADR-0015 — `@cada` benign-read global hygiene, the B16 invariant
tightened to strict name-absence). **R12 is resolved by measurement** — the
re-vectorization peephole is **shelved** and the **matrix-free product family**
accepted as the embedded-efficiency path ([ADR-0016](decisions/ADR-0016-matrix-free-products-efficiency-path.md),
2026-06-26), spawning **R16 (reverse embed-pipeline parity) → R17 (all-axes
showcase + MATLAB→C benchmark) → R18 (matrix-free products H·v/J·v) → R19
(rolled-loop reverse)**, which tie issues #56 / #73 / #64 into one track. **R15
(#64 codegen-equivalence oracle into #38) is approved** — ADR-0014; it is also
the codegen-equivalence + metric infrastructure R17 reuses for its C-level
comparison. **R14, R15, and R16–R19 are authored/verified in the maintainer's
local MATLAB-capable session** (they need MATLAB), so a non-MATLAB session should
not start them. The only maintainer decision still to
surface, not author: **R6** (go/no-go on #6 Tier 2 / #11 Level 3).
**R26 is new** — a batch of core-transform correctness bugs (B17–B22,
`ANALYSIS.md` §1.3c): B17–B21 triaged from a local embedded field report, B22
found during the B17 review. B17 (spurious `.f` on constant-struct fields —
silent broken codegen) is **fixed** (Option-1, pinned by `IConstStructFieldTest`);
the rest triaged (B18 fixed, B22 fixed, B19/B21 open, B20 a documented limitation). Its MATLAB-gated fixes follow the same maintainer-session
rule as R14–R19. **R27 is new** — expand the Monte-Carlo battery to fuzz the full
input-topology × option matrix (#103), motivated by B17/B22 being silent-broken-codegen
bugs the body-only fuzzer never reached.

R7 design note (issue #21): the embedded pipeline already appends the
wrapper, the `_ADiGator*` function, and (inline mode) the data function
into one file (`adigatorGenDerFile_embedded.m`, the
`writelines(...,'WriteMode','append')` block), so the slice runs on the
fully-assembled file with no cross-file boundary — the only
live-by-contract outputs are the entry wrapper's declared returns
(`Jac`/`Hes`/`Grd`/`Fun`). `checkcode` alone is per-function and cannot
link a caller's `dydt.dy_location` read to the subfunction's field
assignment, which is why R7b is the interprocedural field-slice over the
stored function list rather than a single-file lint pass; the
flatten/`checkcode` trick is at most an optional mop-up. The slice-before-
prune ordering (vs. today's prune-first at the top of the per-derivative
loop) is what turns deleted `_location` writes into `.mat`/inline-data
savings.
